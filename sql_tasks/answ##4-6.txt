4. Для безпроблемного удаления записи 
из таблицы offices при наличии связанных записей в таблице employee,
надо в параметре внешних ключей, которые связывают эти две таблицы
в дейтвие по событию ON DELETE установить "CASCADE". Сейчас там "RISTRICT",
т.е., сначала, вручную удаляются все взаимные вхождения таблиц offices и employee,
и только потом, удаляеться сама запись в таблице offices, вцелом.

5. Тип поля в таблице меняеться запросом вида:
	ALTER TABLE tablename MODIFY columnname NEW_TYPE;
	alter table ... change ... - позволяет менять колонку вцелом (имя + тип)
	Можно и более радикально, ALTER TABLE tbl_Country DROP COLUMN + ALTER TABLE ADD, уже с новым типом.
